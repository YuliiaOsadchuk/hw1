Основні команди git шпаргалка

Команда Опис
git config --global user.name blabla встановити ім'я користувача
git config --global user.email blabal@bla встановити email користувача
git init створити новий репозиторій
git status отримати інформацію про стан файлів
git add додати всі змінені файли (до stage)
git commit створити новий коміт
git log показати історію комітів
git checkout переключити репозиторій до вибраного коміту
git branch створити гілку
git push відправка змін у віддалений репозиторій
git pull потягнути зміни з віддаленого репозиторію
git merge з'єднати 2 гілки в одну
git clone клонування існуючого репозиторія
git cherry-pick використовується для того, щоб взяти зміни внесені якимось коммітом і спробувати їх застосувати заново у вигляді нового комміта зверху в поточній гілці.
git rebase "автоматизований" cherry-pick. Він виконує ту ж роботу, але для ланцюжка коммітів, тим самим ніби переносячи гілку на нове місце.
git revert повна протилежність git cherry-pick. Вона створює "антикомміт" для вказанногог комміта, таким чином відміняючи зміни, внесені в ньому.
git reset Скасовує зміни. Змінює вказівник HEAD та опціонально стан індексу. При використанні параметра --hard зміниює файли в робочій директорії.
git stash бере змінені файли, які відслідковуються, та проіндексовані зміни, та зберігає іх в сховище незакінчених змін, котрі можна в будь-який час застосувати знову.
файл .gitignore використовується для вказання в ньому файлів та папок, які необхідно приховати від системи контролю версій git.
Як правило приховують файли або папки, котрі автоматично генерують свій вміст або містять конфігураціонні параметри, які можуть відрізнятися у тих, хто спільно працює над проектом.
